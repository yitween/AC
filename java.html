1. 類別結構與基礎

Java是一個強型別、物件導向的程式語言，由Sun Microsystems（現Oracle）開發，最大特色是「Write Once, Run Anywhere」——Java程式先編譯成「位元組碼（Bytecode）」，再由JVM（Java Virtual Machine）執行，所以可以跨平台運作。Android開發、企業後端系統大量使用Java。

Java程式的每個檔案都必須有一個同名的public class，例如檔案叫Main.java，裡面就要有 public class Main {...}。程式進入點是固定語法：public static void main(String[] args) {...}，一個字都不能差。

Java的輸出用System.out.println()（自動換行）或System.out.print()（不換行），輸入要先建立Scanner物件：Scanner sc = new Scanner(System.in);，然後用 sc.nextInt()讀整數、sc.nextLine()讀字串。

Java的字串用String型態（注意大寫S），字串串接用+；現代Java（15+）支援文字區塊，也可以用String.format()格式化，例如：String.format("我叫%s，今年%d歲", name, age)，格式符和C的printf相同。

Java的「運算子」和C/C++完全相同，加減乘除、%取餘數、比較運算子、邏輯運算子（&&、||、!）都一樣，沒什麼需要特別說明的，直接套用過來用就行！每行敘述結尾要加分號（;）。

2. 變數與資料

Java是強型別語言，所有變數宣告時都必須指定型態。「基本型態（Primitive Types）」有八種：int（整數）、long（大整數）、short（小整數）、byte（更小的整數）、float（單精浮點，後面加f）、double（雙精浮點）、char（字元，用單引號）、boolean（true/false）。

和C#類似，Java也有「包裝類別（Wrapper Classes）」，讓基本型態變成物件，例如int對應Integer、double對應Double，在需要物件的場合（如放進集合）就要用包裝類別。Java會自動在基本型態和包裝類別之間轉換，叫做自動裝箱/拆箱（Auto-boxing）。

Java的「陣列」宣告語法和C#相同：int[] arr = new int[5];，或直接初始化：int[] arr = {1, 2, 3, 4, 5};，索引從0開始，用.length取得長度（注意不是方法，不用加括號）。

更靈活的集合用ArrayList，需要引入java.util.ArrayList。宣告：ArrayList<Integer> list = new ArrayList<>();（注意要用包裝類別），常用方法：.add()新增、.get(i)取值、.set(i, x)修改、.remove(i)刪除、.size()大小。

3. 流程控制

Java的「流程控制」語法和C/C++幾乎完全相同，學過一個就等於學過另一個，這邊快速過一遍。if...else if...else條件判斷、switch多分支（記得加break），Java的switch從Java 14起支援更現代的箭頭語法：case 1 -> ...，不需要break。

迴圈有for、while、do...while，語法和C完全一樣。遍歷陣列或集合可以用「增強型for迴圈（Enhanced for）」，也叫for-each：for(int item : arr){...}，和C#的foreach語法類似，不需要管索引，簡潔易讀。

break跳出迴圈、continue跳過這一輪，用法和C/C++相同。Java還有「帶標籤的break（Labeled break）」，可以跳出指定的外層迴圈，語法是在迴圈前加標籤：outer: for(...){for(...){break outer;}}，在多層巢狀迴圈中很有用。

Java的String比較要用.equals()而不是==，這是Java初學者最常犯的錯誤！因為==比較的是物件的記憶體位址，而.equals()才是比較字串內容。例如：if(name.equals("小明")){...}。

4. 方法

Java把函式叫做「方法（Method）」，所有方法都必須在class內宣告。完整語法是：「存取修飾詞 [static] 回傳型態 方法名稱（參數型態 參數名稱）{... return 值;}」。在main裡呼叫的方法一樣要加static。

「存取修飾詞」有四種：public（任何地方可存取）、private（同class內）、protected（同package和子類別）、無修飾詞（同package，叫package-private）。封裝原則建議屬性設private，再提供public的getter/setter方法來存取。

Java支援「方法多載（Overloading）」，同名方法有不同參數型態或數量。Java是傳值的語言，但傳物件時傳的是參考的副本，所以在方法內修改物件的屬性會影響原物件，但重新指定整個物件（如 obj = new Object()）不會影響原來的參考。

Java也支援「遞迴」，語法和其他語言相同，記得要有終止條件。Java 8之後加入了「Lambda運算式」，語法是 (參數) -> 運算式，例如：Comparator<Integer> cmp = (a, b) -> a - b;，常搭配Stream API做集合操作，是現代Java開發的重要特性。

5. 物件導向核心

Java是「純物件導向語言」（除了基本型態），OOP的四大特性：封裝（把資料和方法包在class內，隱藏實作細節）、繼承（子類別繼承父類別）、多型（同名方法在不同類別有不同行為）、抽象（用介面或抽象類別定義規範），Java把這四個都實作得非常完整。

繼承語法：class Dog extends Animal {...}，Java只支援單一繼承（只能extends一個父類別）。子類別用super()呼叫父類別建構子，用@Override覆寫父類別方法（加這個標註有助於編譯器幫你抓錯）。

多重繼承的需求靠「介面（Interface）」解決，介面用interface宣告，class用implements實作，且可以實作多個介面，例如：class Robot implements Workable, Chargeable {...}。「抽象類別（Abstract class）」用abstract修飾，可以有未實作的方法，介於一般class和interface之間。

Java的「例外處理」用try...catch...finally，語法和C#相同。Java的例外分兩類：checked exception（編譯器強制要處理，例如IOException）和unchecked exception（執行時才發現，例如NullPointerException）。方法若可能拋出checked exception，要在宣告加throws，例如：void readFile() throws IOException {...}。
