1. 網頁的三層結構

製作一個完整的網頁，需要三種技術各司其職：HTML定義內容與結構（這是什麼、放在哪裡）、CSS定義外觀與樣式（看起來怎麼樣）、JavaScript定義行為與互動（點了之後發生什麼）。三者缺一不可，好的網頁需要三者平衡配合。

一個網頁的標準檔案結構：建立一個資料夾，裡面放index.html（首頁）、style.css（樣式）、script.js（腳本），圖片放在images/子資料夾。HTML裡用<link rel="stylesheet" href="style.css">引入CSS，用<script src="script.js" defer></script>引入JS（defer讓JS在HTML載入完後才執行，避免找不到元素的問題）。

HTML的「語意結構」是現代網頁開發的基礎：用<header>放Logo和導覽列、<main>放主要內容、<section>分區塊、<article>放獨立文章、<footer>放頁尾資訊。語意標籤除了讓程式碼更易讀，對SEO（搜尋引擎優化）也很有幫助。

CSS的「變數（Custom Properties）」讓你建立設計系統：在:root裡定義全局變數，例如：--primary-color: #00ffcc;，之後用color: var(--primary-color);引用，要改主題色只需改一個地方，整站都更新，是大型專案的必備技巧。

JS的「模組化」讓程式碼可維護：把不同功能拆成不同的.js檔，用export和import管理。例如：utils.js放工具函式、api.js放所有API呼叫、ui.js放DOM操作——這樣找bug和修改功能時，知道要去哪個檔案，不會在幾千行的大檔裡迷路。

2. 響應式版面設計

現代網頁必須在手機、平板、桌機上都好看，這就是「響應式設計（Responsive Web Design）」。核心思路：先設計手機版（Mobile First），再用Media Query逐步擴展到大螢幕，例如：@media (min-width: 768px)套用平板樣式，@media (min-width: 1200px)套用桌機樣式。

Flexbox做元件排版：導覽列、卡片清單、按鈕群組都很適合。父元素設display: flex後，用flex-wrap: wrap讓子元素在空間不足時自動換行，例如每張卡片設flex: 1 1 300px（可縮放、可換行、最小寬300px），畫面寬時三欄並排，窄時自動變成單欄。

CSS Grid做整頁版面配置：grid-template-columns: 1fr 3fr做側邊欄+主內容；grid-template-areas可以直觀地命名各區塊，例如："header header" "sidebar main" "footer footer"，然後子元素用grid-area: header指定歸屬，排版邏輯一目瞭然。在手機版用Media Query把grid改成grid-template-columns: 1fr就變成單欄。

圖片響應式：img { max-width: 100%; height: auto; }讓圖片隨容器縮放，不超出邊界；用<picture>標籤搭配srcset，讓瀏覽器根據螢幕選擇合適尺寸的圖片，手機不用下載桌機版的大圖，加快載入速度。

字型和間距用rem為單位（相對於html的font-size，預設16px），而非px，讓使用者在瀏覽器調整字型大小時整頁等比縮放。clamp(最小值, 理想值, 最大值)讓字型大小在範圍內根據視窗寬度流暢縮放，例如：font-size: clamp(1rem, 2.5vw, 2rem)，這是現代CSS的優雅解法！

3. JavaScript 互動效果

靜態網頁加上JS就能活起來！最基本的互動：「導覽列」——監聽scroll事件，捲動超過一定距離時加上class讓navbar背景變深；漢堡選單按鈕切換側邊選單的顯示/隱藏（用classList.toggle()）。程式碼：document.querySelector('.menu-btn').addEventListener('click', () => { document.querySelector('.nav').classList.toggle('open'); })。

「捲動動畫」：元素進入視窗時才執行動畫，用Intersection Observer API，比監聽scroll事件更高效：new IntersectionObserver((entries) => { entries.forEach(e => { if(e.isIntersecting) e.target.classList.add('visible') }) }).observe(element)，配合CSS的transition就能做出漂亮的淡入效果。

「表單驗證」：在送出前用JS驗證輸入，提供比原生HTML驗證更好的使用者體驗：即時驗證（input事件）、顯示自訂錯誤訊息、email格式用正則表達式檢查（/^[^\s@]+@[^\s@]+\.[^\s@]+$/）、密碼強度檢測。驗證不通過就阻止送出：e.preventDefault()。

「深色/淺色模式切換」：讀取localStorage記住使用者偏好，切換時對document.body加上class="dark"，CSS裡用.dark選取器覆蓋CSS變數即可，例如：.dark { --bg-color: #141414; --text-color: #eee; }。這是現代網頁的標配功能。

還可以用fetch()打API取得動態資料，例如從公開API載入天氣、匯率、新聞等，讓你的靜態網頁也能顯示即時資訊。JSON資料用response.json()解析，再用JS動態建立DOM元素插入頁面，這就是現代「單頁應用程式（SPA）」的基礎概念。

4. 網頁效能與優化

網頁載入速度直接影響使用者體驗和SEO排名。用Chrome開發者工具的Lighthouse跑分析，它會給你Performance、Accessibility、SEO等項目的評分，並列出改善建議，是優化網頁最好的起點。

最大的效能殺手通常是「圖片」：改用WebP格式（比JPEG/PNG小30-80%）；用線上工具或squoosh.app壓縮圖片；設定loading="lazy"讓圖片延遲載入（捲到才載入）；用CSS的object-fit: cover控制圖片填充方式，不要用JS動態縮放圖片。

CSS和JS的載入優化：CSS放在<head>裡（阻擋渲染，讓頁面一開始就有樣式）；JS加defer（等HTML解析完再執行）或async（異步下載，下載完立即執行）；把多個CSS/JS檔案合併並壓縮（Minify）成一個，減少HTTP請求數量；用CDN加速靜態資源分發。

CSS動畫效能：優先使用transform和opacity做動畫，這兩個屬性由GPU加速，不觸發頁面重排（reflow），非常流暢；避免動畫width、height、margin等會引起重排的屬性，會讓頁面卡頓。will-change: transform提示瀏覽器提前做GPU加速準備。

「無障礙設計（Accessibility）」讓所有人都能使用你的網頁：圖片加alt屬性（讓螢幕閱讀器描述圖片）；按鈕和連結有清楚的文字；表單用<label>關聯input；確保色彩對比度足夠；網頁可以只用鍵盤瀏覽（Tab切換焦點）。這不只是做好事，也是法規要求的方向！

5. 實作專案：個人作品集網站

整合所學，做一個「個人作品集網站（Portfolio）」！這是展示你的技術能力最直接的方式，也是求職的重要加分項。規劃頁面區塊：Hero（大標題+自我介紹）、About（關於我）、Skills（技能清單）、Projects（作品展示）、Contact（聯絡方式），每個區塊對應一個<section>。

Hero區塊：用CSS背景漸層（linear-gradient）或SVG圖案做背景；大標題用CSS動畫做打字機效果（用JS在setInterval裡逐字顯示）；CTA按鈕做hover時的滑動背景效果（用::before偽元素+transform）。

作品卡片：用CSS Grid做卡片牆，每張卡片hover時輕微上移（transform: translateY(-8px)+box-shadow加深），點擊展開Modal顯示作品詳情，Modal用position: fixed全螢幕遮罩。用JS的fetch()從GitHub API自動拉取你的專案資料（api.github.com/users/你的名字/repos），不用手動更新！

聯絡表單：用Formspree或EmailJS等免費服務，讓靜態網頁也能發送email，不需要後端。JS監聽表單送出，用fetch呼叫服務API，成功後顯示感謝訊息，失敗時顯示錯誤。

部署：把網站上傳到GitHub Pages或Netlify就能免費公開！GitHub Pages直接從你的Repository部署，Netlify更強大，支援自動從GitHub更新、自訂域名、HTTPS。讓全世界都能看到你的作品，這就是製作網頁最令人興奮的時刻！🌐
