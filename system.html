1. Linux 基礎回顧

要用程式做「系統」，首先要了解「作業系統（OS）」本身。Linux是目前伺服器和嵌入式裝置最主流的OS，學習在Linux上製作系統工具，是工程師的必備技能。這章以Linux指令和Shell Script為核心，帶你做出真實可用的系統管理工具。

系統開發最常用的Linux指令：ps aux查看所有行程（Process）；top/htop即時監控CPU、記憶體；df -h查磁碟使用量；free -h查記憶體用量；netstat -tulnp（或ss -tulnp）查看開放的網路埠口；lsof查哪個程式開了哪些檔案。

系統日誌（Log）是診斷問題的重要線索，存放在/var/log/目錄：syslog或messages是系統總日誌；auth.log記錄登入驗證；kern.log記錄核心訊息。查看日誌用tail -f /var/log/syslog即時追蹤，或用journalctl -xe（systemd系統）查看詳細錯誤。

Linux上的「環境變數（Environment Variables）」是系統設定的重要方式：echo $PATH查看執行檔搜尋路徑；export MY_VAR="value"設定環境變數（只在當前session有效）；要永久保留則加到~/.bashrc或~/.profile，然後source ~/.bashrc讓它生效。環境變數讓你的程式可以讀取系統設定，不用把密碼或路徑寫死在程式碼裡。

最後，「管道（Pipe）」和「重定向」是組合指令的核心：|把上一個指令的輸出傳給下一個；grep過濾內容；awk擷取特定欄位；sort排序；uniq去重；例如：ps aux | grep python | awk '{print $2}' 就能列出所有Python行程的PID。

2. Shell Script 程式設計

「Shell Script」是Linux系統自動化的核心，把一連串指令寫成腳本，一鍵執行複雜任務。建立Script：新增副檔名.sh的檔案，第一行固定寫#!/bin/bash（Shebang），告訴系統用bash執行，然後chmod +x script.sh賦予執行權，用./script.sh執行。

Shell Script的「變數」不需要宣告型態，直接賦值：NAME="Alice"（等號兩邊不能有空格！），使用時加$：echo $NAME。接收命令列參數：$1是第一個參數、$2是第二個，$#是參數總數，$@是所有參數。用read 變數名可以互動式地讀取使用者輸入。

流程控制：條件判斷用if [ 條件 ]; then ... elif ... else ... fi，條件裡數字比較用-eq（等於）、-ne（不等於）、-gt（大於）、-lt（小於）；字串比較用=和!=；判斷檔案存在用-f 檔名，判斷目錄存在用-d 目錄名。

迴圈方面：for i in {1..10}; do ... done跑1到10；for file in *.log; do ... done遍歷所有log檔；while [ 條件 ]; do ... done條件迴圈；用$(指令)把指令的輸出存成變數，例如：FILES=$(ls *.txt)。函式定義：function_name() { ... }，呼叫：function_name 參數。

實用技巧：set -e讓Script在任一指令失敗時立即停止（很重要！）；set -x開啟debug模式（每行執行前先印出）；trap 'cleanup' EXIT讓Script結束時（不管正常或異常）執行清理函式；$?取得上一個指令的結束碼（0=成功，非0=失敗）。

3. 系統服務管理

現代Linux用「systemd」管理系統服務（daemon），讓你的程式可以開機自動啟動、崩潰時自動重啟。查看服務狀態：systemctl status nginx；啟動/停止/重啟：sudo systemctl start/stop/restart nginx；設為開機自啟：sudo systemctl enable nginx；查看所有服務：systemctl list-units --type=service。

要把自己的程式做成系統服務，需要建立一個「.service 設定檔」，放在/etc/systemd/system/下，例如 myapp.service，基本內容包含：[Unit]描述服務、[Service]設定執行指令（ExecStart）、重啟策略（Restart=always）、執行使用者（User）、[Install]設定啟動等級。設好後執行sudo systemctl daemon-reload讓systemd重新讀取設定。

「行程管理」是系統開發的重要技能：kill -9 PID強制終止行程；killall 程式名終止所有同名行程；nohup 指令 &讓程式在背景持續運行，關掉終端也不會停；screen或tmux可以建立可以保留的終端會話，是遠端工作的必備工具。

系統資源限制用ulimit設定，例如：ulimit -n 65536提高可開啟檔案數（預設1024，做高流量伺服器時常需要調整）。nice -n 10 ./heavy_job可以降低行程的CPU優先權，讓系統其他程式不被拖累；renice可以對已執行中的行程調整優先權。

網路設定方面：ip addr show查看網路介面和IP；ip route show查看路由表；iptables或ufw設定防火牆規則，例如sudo ufw allow 80/tcp開放80埠。這些都是架設真實系統服務的必要知識。

4. 排程與系統監控

「Cron排程」讓你的系統任務定時自動執行，用crontab -e編輯，格式是「分 時 日 月 週 指令」，例如：0 2 * * * /home/user/backup.sh每天凌晨2點執行備份；*/5 * * * * /home/user/monitor.sh每5分鐘執行一次監控腳本；0 0 1 * * /home/user/monthly.sh每月1號執行。crontab -l查看目前設定。

系統監控腳本的核心是把各種系統資訊蒐集、判斷、通知。例如：用df -h | awk擷取磁碟使用率，超過80%就發警告；用free | awk計算記憶體使用百分比；用ping -c 1 目標IP檢查網路連通性，返回碼非0就代表不通。把這些判斷寫進Shell Script，配上Cron定時執行，就是基本的系統監控工具。

通知機制：最簡單的是用mail指令發email；或呼叫「Webhook」，例如Slack、LINE Notify的API，用curl -X POST -d "message=警告" https://notify-api.line.me/api/notify送出通知訊息，讓你在手機上即時收到系統警報。

「Log管理」是系統穩定運作的關鍵：用logrotate自動輪替日誌（避免log檔無限成長撐爆磁碟）；在Script中把輸出寫入log：echo "$(date) 備份完成" >> /var/log/myapp.log；$(date +"%Y-%m-%d %H:%M:%S")取得當前時間戳記，讓log有時間記錄。

進階監控可以用Python搭配psutil套件，提供更豐富的系統資訊API：CPU、記憶體、磁碟、網路流量全都可以用Python程式讀取和分析，比Shell Script更靈活，也更容易整合資料庫儲存歷史數據。

5. 實作專案：系統監控儀表板

把前面學到的所有技術整合起來，做一個「系統健康監控儀表板」！整體架構：Shell Script定期收集系統資訊 → 寫入文字檔或資料庫 → Python Flask提供Web API → 簡單的HTML/JS前端顯示即時圖表。這是一個完整的全端系統小專案！

資料收集腳本（collect.sh）：每分鐘執行，蒐集CPU使用率（top -bn1 | grep "Cpu(s)"）、記憶體（free -m）、磁碟（df -h /）、目前時間，格式化成CSV追加寫入metrics.csv，用Cron設定每分鐘執行。

Python API（app.py）：用Flask建立簡單後端，讀取metrics.csv的最近N筆資料，提供/api/metrics端點回傳JSON；若有告警條件（例如CPU>90%）則在回傳資料中標記。flask run --host=0.0.0.0啟動後，外部機器就能透過HTTP存取。

前端頁面：用HTML+JS的Chart.js每10秒呼叫一次API，把回傳的數據畫成即時折線圖；有告警時用紅色顯示。這樣就完成了一個可以在瀏覽器上看到伺服器即時狀態的監控系統！

最後，把Flask app做成systemd服務，確保主機重開機後自動啟動；把collect.sh設定Cron排程。這樣整個系統就會自動持續運作，不需要手動維護，這就是系統自動化的精髓！恭喜完成一個真實的系統工程專案！🖥️
