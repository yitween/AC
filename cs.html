1. 環境與基礎

C#（唸「C Sharp」）是微軟開發的物件導向語言，執行在「.NET 平台」上，和Java有點像，都是先編譯成中間碼再由虛擬機執行，讓程式可以跨平台運作。C#語法大量借鑒了C++，但更簡潔現代，拿來開發Windows應用、遊戲（Unity就是用C#）、後端服務都非常適合。

C#程式的基本結構長這樣：最外層用namespace包住（命名空間，用來組織程式碼，避免名稱衝突）；裡面宣告一個class；再裡面是static void Main(string[] args)，這是程式的進入點，對應C語言的int main()，一切從這裡開始跑。

C#的輸出用Console.WriteLine()（會自動換行）或Console.Write()（不換行），例如 Console.WriteLine("Hello World");。輸入用Console.ReadLine()，它回傳的是string，若要轉成數字需要用int.Parse()或Convert.ToInt32()。

C#最方便的是「字串插值（String Interpolation）」，在字串前加$就可以在雙引號內用{變數名稱}直接嵌入變數值，例如 Console.WriteLine($"我叫{name}，今年{age}歲");，比起C的printf格式符直觀很多！

另外，C#的每一行敘述也要以分號（;）結尾，程式區塊用大括號{}包住，和C/C++一樣。C#也有using關鍵字（類似C++的#include），例如using System;可以引入基礎函式庫，這樣才能用Console等常用工具。

2. 變數與資料

C#的「資料型態」分為實值型態（Value Types）和參考型態（Reference Types）兩大類。實值型態直接儲存資料本身，參考型態儲存的是指向資料的位址，這個概念之後學到class時會更清楚。

常見的實值型態有：int（整數）、long（大整數）、float（單精度浮點數，後面要加f，例：3.14f）、double（雙精度浮點數）、decimal（超高精度，適合金融計算）、char（字元，用單引號）、bool（true/false）。常見的參考型態有string（字串）和陣列。

C#有個超方便的關鍵字var，讓編譯器自動推斷型態，例如 var x = 10; 編譯器會自動知道x是int。這讓程式碼更簡潔，但可讀性有時會下降，建議型態明確時還是寫清楚比較好。

C#的「陣列」宣告語法是「型態[] 名稱 = new 型態[大小];」，例如 int[] arr = new int[5];，或直接初始化 int[] arr = {1, 2, 3, 4, 5};。索引同樣從0開始，陣列有個.Length屬性可以取得長度，比C語言方便很多！

更強大的是List<T>，類似C++的vector，是動態大小的集合，需要引入System.Collections.Generic。宣告：List<int> list = new List<int>();，新增用.Add()，移除用.Remove()，數量用.Count，比陣列靈活超多！

3. 流程控制

C#的「流程控制」和C/C++非常相似，有條件判斷和迴圈兩大類，語法幾乎一樣，但C#多了一些更現代的寫法，讓程式碼更簡潔。

條件判斷用if...else if...else，語法和C++相同。C#的switch更強大，不只能比對整數，字串、enum甚至型態都可以比對，例如 switch(name){ case "Alice": ...; break; }，和C一樣記得加break。

C#還有個超簡潔的「三元運算子」，語法是「條件 ? 成立的值 : 不成立的值」，例如 int max = (a > b) ? a : b;，一行搞定if-else，適合簡單的賦值判斷。

迴圈一樣有for、while、do...while，語法和C++相同。C#還多了一個很好用的foreach，專門用來遍歷集合，語法是 foreach（int item in arr）{...}，不用管索引，直接一個一個把元素拿出來用，遍歷陣列或List時非常方便！

同樣地，break用來跳出迴圈，continue用來跳過這一輪直接進行下一輪。C#還有return可以直接結束整個方法，在複雜邏輯中「提早return」是讓程式碼更清晰的好習慣，比一層一層if-else好讀多了。

4. 方法

C#把函式稱為「方法（Method）」，因為在C#裡所有東西都必須在class裡面，所以函式的正式名稱就叫方法。宣告語法是：「存取修飾詞 回傳型態 方法名稱（參數型態 參數名稱）{... return 值;}」。

「存取修飾詞」決定方法可以被誰呼叫：public（任何地方都能用）、private（只有同一class內能用）、protected（同class和子class能用）。在Main裡直接呼叫的方法需要加static，因為Main本身是static的，static方法可以不用建立物件就直接呼叫。

C#的「參數傳遞」預設也是傳值，若要修改原變數，可以用ref（傳參考，變數要先初始化）或out（也是傳參考，但方法內必須賦值，常用在一個方法要回傳多個值的情況），例如：void GetInfo(out int age, out string name){age = 18; name = "Alice";}。

C#支援「方法多載（Overloading）」，同名方法可以有不同的參數型態或數量，編譯器自動選對的版本。另外還有「選擇性參數」，在宣告時給預設值，呼叫時可以省略，例如：void Greet(string name = "訪客"){...}，這樣呼叫Greet()不傳參數也沒問題！

C#還有個強大的「Lambda 運算式」，可以用超簡短的語法定義小型函式，符號是=>，例如：var square = (int x) => x * x;，呼叫 square(5) 就回傳25。搭配LINQ和集合操作時非常好用，是現代C#開發的必學語法。

5. 物件與特性

C#是完全的「物件導向語言（OOP）」，這章是整個C#的核心精神。先說「類別（class）」：它是物件的設計圖，裡面有屬性（Properties）儲存資料和方法（Methods）定義行為，而「物件（object）」就是依照設計圖建出來的實體，用new關鍵字建立，例如：Dog myDog = new Dog();。

C#的「屬性（Property）」比C++的成員變數更強大，可以加上get和set存取子來控制讀寫，例如：public int Age { get; set; }，這叫「自動屬性」，也可以在get/set裡加邏輯，例如防止年齡被設為負數。

「建構子（Constructor）」和class同名、沒有回傳型態，建立物件時自動執行，用來初始化屬性，可以多載。「繼承（Inheritance）」語法是 class 子類別 : 父類別 {...}，子類別會繼承父類別的public和protected成員，可以用override覆寫父類別的virtual方法，這就是「多型（Polymorphism）」。

C#還有「介面（Interface）」，用interface關鍵字宣告，只定義方法的簽名而不實作，讓實作它的class保證有這些方法，例如：interface IAnimal { void Speak(); }。一個class可以實作多個interface，解決了C#不支援多重繼承的限制。

最後介紹C#超實用的「例外處理」，語法和C++類似：try {...} catch（Exception e）{...} finally {...}，finally區塊不管有沒有例外都會執行，通常用來做清理工作（例如關閉檔案）。可以用throw new Exception("錯誤訊息")手動拋出例外，也可以用更具體的型態如ArgumentNullException、IndexOutOfRangeException等讓錯誤更清楚。
