COMPETITION

1. 競賽程式概論

「競賽程式設計（Competitive Programming）」是在限定時間內解決演算法問題的競技，考驗邏輯推理、資料結構和演算法的掌握程度。主要競賽：APCS（大學入學先修測驗）、全國高中資訊奧林匹亞（TOI）、ICPC（國際大學生程式競賽）、LeetCode/Codeforces（線上練習平台）。

語言選擇：C++是競賽的主流語言，速度快、STL提供豐富資料結構、大多數競賽都支援；Python語法簡潔，適合快速實作，但速度較慢，時間限制較嚴的題目可能超時；Java速度介於C++和Python之間，BigInteger處理大數很方便。多數競賽選手以C++為主，Python/Java作為輔助。

C++競賽常用的開頭：#include <bits/stdc++.h>（引入所有標準函式庫）；using namespace std;；typedef long long ll;（ll比int大，避免overflow）；ios_base::sync_with_stdio(false); cin.tie(NULL);（加速輸入輸出，大量IO時很重要）。

解題的基本流程：讀清楚題目（輸入格式、限制條件、輸出要求）→ 分析規律找思路 → 考慮時間複雜度（N=10^5的題目通常要O(N log N)以內）→ 寫程式 → 手動測試基本案例和邊界案例（空、單元素、最大值）→ 提交。Time Limit Exceeded（TLE）通常是演算法複雜度太高；Runtime Error（RE）通常是陣列越界或除以0；Wrong Answer（WA）是邏輯錯誤。

心態：競賽題目設計來讓你挑戰極限，看不懂或想不出來是正常的！重要的是每次解完題要看其他人的解法，了解更好的思路；累積「見過的題型」，看到新題目才能聯想到解法。練習量是進步的關鍵，每天穩定練習比突擊更有效。

2. 基礎資料結構

競賽中最常用的「資料結構」都在C++ STL裡：vector（動態陣列）、stack（後進先出）、queue（先進先出）、priority_queue（堆積，自動維護最大/最小值）、set（有序不重複集合，O(log N)操作）、map（有序鍵值對，O(log N)）、unordered_map（雜湊表，O(1)平均）。

「堆疊（Stack）」的應用：括號配對（遇到左括號push，遇到右括號檢查pop的是否配對）、計算後綴表達式、DFS的迭代版本、單調堆疊（找每個元素左/右第一個更大的元素，O(N)）。「佇列（Queue）」的應用：BFS層序遍歷、滑動視窗最大值（用deque）。

「二元樹」：理解前/中/後序遍歷（DFS，用遞迴或堆疊）和層序遍歷（BFS，用佇列）。「二元搜尋樹（BST）」讓搜尋/插入/刪除都是O(log N)；STL的set和map內部用紅黑樹實作，是平衡BST，保證最差O(log N)。「Heap（堆積）」快速取最大/最小，priority_queue就是max-heap；自訂比較函式：priority_queue<int, vector<int>, greater<int>>是min-heap。

Python的對應：list當stack（append/pop）；collections.deque當queue（appendleft/pop）；heapq模組做min-heap；collections.defaultdict和Counter很好用；Java的對應：Stack、LinkedList（當deque）、PriorityQueue（min-heap）、TreeMap/TreeSet、HashMap/HashSet。

3. 核心演算法

「排序與搜尋」：sort(v.begin(), v.end())（C++ STL，O(N log N)）；自訂排序：sort(v.begin(), v.end(), [](int a, int b){ return a > b; })（降序）；binary_search/lower_bound/upper_bound在排序陣列上O(log N)搜尋。二分搜尋的模板要背熟，很多題目用「二分答案」技巧（對答案二分，check函式判斷可行性）。

「圖論」是競賽必考：圖的表示用鄰接表（Adjacency List），C++用vector<vector<pair<int,int>>> adj(n)；BFS（廣度優先搜尋）：用queue，找最短路徑（無權圖）；DFS（深度優先搜尋）：用遞迴或stack，找連通塊、拓撲排序；Dijkstra：有權圖最短路，用priority_queue，O((V+E)log V)；Floyd-Warshall：所有點對最短路，O(V³)。

「動態規劃（DP）」是競賽最重要也最難的技巧，把大問題分解成子問題、記憶化避免重複計算。入門題型：費氏數列（dp[i] = dp[i-1]+dp[i-2]）→ 爬樓梯；0/1背包問題（dp[i][w] = max選不選第i件物品）；最長公共子序列（LCS）；最長遞增子序列（LIS）。DP的關鍵是定義「狀態」和「轉移方程式」，把思路寫清楚再碼。

常用技巧：前綴和（Prefix Sum）——O(1)查詢任意區間和；差分陣列——O(1)區間更新；Union-Find（並查集）——快速判斷兩點是否連通、合併集合（Kruskal最小生成樹用到）；位元運算——用XOR找只出現一次的數、用AND/OR做狀態壓縮DP。

4. APCS 考試解題策略

APCS（大學程式設計先修檢測）是台灣高中生最重要的程式競賽考試，分為「觀念題」（選擇題，考基礎概念）和「實作題」（程式設計，提交程式碼評分）。觀念題考變數、迴圈、遞迴、資料結構等基本觀念，要熟悉各種程式追蹤（看一段程式推輸出）；實作題共4題，難度從1~5，建議先把能做的題目做完，再攻難題。

APCS常見題型：模擬題——按照題目規則一步一步執行，忠實實作即可；字串處理——切割、比對、計數；陣列操作——前綴和、排序、二分；簡單圖論——BFS/DFS連通塊；基礎DP——背包、爬樓梯類型。3、4級對學測加分有利，5級需要深厚的演算法功底。

時間管理策略：每題先看題目和subtask（子題）分數配置，有些題目部分分數只需要O(N²)暴力解就夠了；先確保拿到每題的基本分，再挑戰難的部分；不要在一題卡太久，先跳過去做後面的題目。

Codeforces/LeetCode練習策略：每次做題前不要看提示，想不出來時再看（think first）；做完後一定要看別人的解法（editorial），了解更優雅的思路；建立自己的「題型筆記」，把每種題型的解法模板記下來；從Easy/CF Rating 800開始，每週挑戰高一點的難度，循序漸進。

Python在競賽中的加速技巧：sys.stdin.readline比input()快很多（大量輸入時必用）；sys.setrecursionlimit(10**6)增加遞迴深度限制；PyPy（多數OJ支援）比CPython快3-5倍；用collections.Counter、defaultdict替代手動計數；避免在迴圈內做字串拼接（改用join）。

5. 進階演算法與備賽

進入TOI/ICPC等頂尖競賽需要更深的演算法知識。「線段樹（Segment Tree）」：支援O(log N)的區間查詢（max/min/sum）和單點/區間修改，是很多進階題的標配；「樹狀陣列（BIT / Fenwick Tree）」：實作更簡單，支援前綴和查詢和單點修改，O(log N)；兩者都比暴力的O(N)查詢快很多，在N=10^5的題目中是關鍵。

「最短路進階」：Bellman-Ford（可處理負邊，O(VE)，偵測負環）；SPFA（Bellman-Ford的佇列優化，競賽中常被卡）；Johnson算法（全點對最短路，適合稀疏圖）。「最小生成樹」：Kruskal（排序邊，Union-Find合併）；Prim（類似Dijkstra，從一點出發擴張）。

「進階DP」：樹狀DP（在樹上做DP，根為子問題）；狀態壓縮DP（用bitmask表示狀態，例如TSP旅行商問題）；區間DP（dp[l][r]代表區間l到r的最優解）；斜率優化DP（高級優化技巧，把O(N²) DP降到O(N)）。

數論：GCD（輾轉相除法）；質數篩（Sieve of Eratosthenes）；模運算（__int128處理超大數，或用取模防溢出）；快速冪（Fast Exponentiation）計算a^b mod p，O(log b)。字串：KMP演算法（字串比對O(N+M)）；Z-function；字典樹（Trie）高效前綴搜尋。

備賽計畫建議：至少提前3個月開始有系統地練習；用vjudge建立虛擬比賽模擬真實考試感；每週至少做一場Codeforces虛擬比賽；把錯的題目整理成錯題本定期複習；找志同道合的同學組隊練習，互相講解思路是最好的學習方式。競賽之路漫長但充實，每次進步都是自己和過去的自己的競爭！💪
