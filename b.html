1. 後端是什麼

「後端（Backend）」是使用者看不見的部分——伺服器、資料庫、API——負責處理業務邏輯、儲存資料、和前端溝通。前端負責「呈現」，後端負責「運算和儲存」，兩者透過HTTP API溝通，這就是現代Web應用的基本架構。

「API（Application Programming Interface）」是前後端的橋樑，最常見的是REST API：用不同的HTTP方法對應不同操作——GET（查詢）、POST（新增）、PUT/PATCH（修改）、DELETE（刪除）；資料格式用JSON傳遞；URL設計語意清楚，例如/api/users/123代表id=123的使用者。

後端開發最常用的資料庫是「關聯式資料庫（RDBMS）」：MySQL（最廣泛使用）、PostgreSQL（功能更強，支援JSON欄位、地理資訊等）、SQLite（輕量，適合小型專案和開發測試）。資料以表格（Table）形式儲存，表格之間靠外鍵（Foreign Key）關聯，用SQL語言操作。

非關聯式資料庫（NoSQL）在某些場景更適合：MongoDB（文件型，存JSON，schema彈性）適合資料結構多變；Redis（鍵值型，記憶體儲存，超快速）適合快取（Cache）、Session儲存、排行榜；選什麼資料庫要根據專案需求決定，沒有絕對的好壞。

後端的三層架構：路由層（Router）接收請求並分配到對應的處理函式；業務邏輯層（Service）處理核心功能；資料存取層（Repository/DAO）負責和資料庫溝通。這種分層設計讓程式碼職責清楚、容易測試和維護，是企業級後端的標準結構。

2. 資料庫設計與 SQL

好的「資料庫設計」是後端的根基。設計原則：正規化（Normalization）——避免資料重複，每個資訊只存一個地方，用外鍵關聯；每張表加上id作為主鍵（Primary Key），通常設為auto-increment；時間欄位加created_at和updated_at，追蹤資料的建立和修改時間。

資料表關係有三種：一對多（1:N）——一個使用者有多篇文章（users.id ← posts.user_id）；多對多（M:N）——學生和課程，需要第三張「中間表」（student_courses）；一對一（1:1）——使用者和個人資料，通常可以合併成一張表，除非有效能考量才分開。

後端常用的SQL進階技巧：JOIN連接多張表查詢；INDEX（索引）加在常被WHERE查詢的欄位，大幅提升查詢速度（例如CREATE INDEX idx_email ON users(email);）；Transaction（交易）確保多個SQL操作全部成功或全部回滾，例如轉帳：扣款和入帳必須同時成功，用BEGIN; ... COMMIT;或ROLLBACK;。

「ORM（Object-Relational Mapping）」讓你用程式語言操作資料庫，不用直接寫SQL。Python的SQLAlchemy（或Django ORM）讓你用Python class代表資料表，用物件的方式新增查詢，例如：User.query.filter_by(email='a@b.com').first()。ORM更安全（自動防範SQL Injection）、更好維護，是現代後端的主流做法。

SQL資安：「SQL Injection」是最常見的資安漏洞，攻擊者透過輸入惡意SQL破壞資料庫。永遠不要用字串拼接SQL：f"SELECT * FROM users WHERE name='{user_input}'"，這非常危險！要用參數化查詢（Parameterized Query）：cursor.execute("SELECT * FROM users WHERE name=?", (user_input,))，或直接用ORM。

3. 後端框架：Flask 與 Node.js

Python的Flask是輕量後端框架，適合學習和小型專案。安裝：pip install flask。最簡單的Flask應用：from flask import Flask, jsonify, request；app = Flask(__name__)；@app.route('/api/hello', methods=['GET'])裝飾器定義路由；函式裡回傳jsonify({'message': 'Hello'})；最後app.run(debug=True)啟動。

Flask處理不同HTTP方法：request.args.get('key')取URL參數（GET的?key=value）；request.json取POST/PUT的JSON body；request.headers取header（例如Authorization token）。回應狀態碼：return jsonify(data), 201（創建成功）；return jsonify({'error': 'Not found'}), 404。

「使用者認證（Authentication）」是後端核心功能：常用JWT（JSON Web Token），使用者登入後伺服器產生一個Token，之後每次請求把Token放在Header（Authorization: Bearer <token>），伺服器驗證Token的有效性。密碼絕對不能明文儲存，要用bcrypt做雜湊（Hash）再存到資料庫。

「中介層（Middleware）」讓你在請求到達路由之前做統一處理：身份驗證（每個需要登入的路由都要檢查Token，用middleware一次處理，不用每個函式都寫）；CORS設定（允許前端跨域存取API，用flask-cors套件）；請求Log記錄（每次API呼叫都記下時間、路徑、狀態碼）。

Node.js的Express是JS後端的主流選擇，語法和Flask類似：app.get('/api/users', (req, res) => { res.json(users) })。Node.js的異步特性讓它在高並發場景效能優異，搭配MongoDB是MEAN/MERN Stack的組合，很多新創公司的技術棧。

4. 伺服器部署與維運

程式寫好了，要把它部署到「伺服器」上才能讓別人存取。常見的雲端平台：Heroku（免費方案易用，適合學習）、Railway / Render（現代替代方案）、AWS / GCP / Azure（企業級，功能最強但複雜）、DigitalOcean（便宜、開發者友善）。

生產環境的Flask不應該用內建的開發伺服器，要用Gunicorn（WSGI伺服器）：pip install gunicorn；啟動：gunicorn -w 4 -b 0.0.0.0:8000 app:app（4個worker行程，監聽8000埠）。前面再加一個Nginx反向代理，處理靜態檔案、SSL終止、負載平衡，這是業界標準的部署架構。

「Docker」讓部署更標準化：把應用程式和所有依賴打包成容器（Container），「在我電腦上跑得好好的」的問題就消失了——任何有Docker的機器都能跑一樣的容器。建立Dockerfile定義容器內容，docker-compose.yml同時啟動後端app + 資料庫 + Redis的整組服務，是現代開發的標準工作流程。

資料庫維護：定期備份（Backup）——mysqldump -u root -p mydb > backup.sql，配合Cron排程自動每天備份；資料量大時要做讀寫分離（主庫寫、從庫讀）或分片（Sharding）；用Migration工具（Flask-Migrate、Alembic）管理資料庫schema版本，不要直接手動改生產資料庫！

後端監控：Sentry自動捕捉程式例外錯誤並發警報；Prometheus + Grafana監控API回應時間、錯誤率、資料庫查詢速度等指標；設定適當的Health Check端點（/api/health），讓負載平衡器知道服務是否正常。這些是進入企業後端開發必學的工具。

5. 實作專案：RESTful API 後端

做一個完整的「RESTful API」後端！題材：「任務管理系統」——使用者可以註冊登入、建立任務、標記完成、查詢自己的任務清單。這包含了後端最核心的CRUD和認證功能。

API設計：POST /api/auth/register（註冊）、POST /api/auth/login（登入，回傳JWT token）；GET /api/tasks（查詢我的任務）、POST /api/tasks（新增任務）、PUT /api/tasks/:id（修改任務）、DELETE /api/tasks/:id（刪除任務）。除了auth，其他端點都需要JWT驗證。

資料庫設計：users表（id, email, password_hash, created_at）；tasks表（id, user_id, title, description, is_done, due_date, created_at, updated_at），user_id是外鍵關聯users。用SQLAlchemy定義Model，Flask-Migrate管理migration。

用Postman或Insomnia測試API：可以儲存所有的測試請求、設定環境變數（把Token存起來，所有請求自動帶上）、建立自動化測試集，是後端開發的重要工具。寫完API後也可以用pytest寫自動化測試，確保功能正確、之後改Code時不會壞掉。

完成後部署到Railway或Render（免費方案可用），讓前端夥伴能實際連線測試。把API文件用Swagger / OpenAPI（Flask用flask-restx自動產生）整理好，讓別人知道怎麼用你的API，這是後端工程師的專業素養！🚀
